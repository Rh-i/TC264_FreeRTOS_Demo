## 21届智能车规则

### 赛道

车辆在真实物理赛道内行驶，同时系统会实时叠加虚拟行人、虚拟道路标识、虚拟障碍物、虚拟目标物（如金币、特效奖励、任务道具）等信息。增强现实系统由百度提供。

> 我们需要注意的是：这个**物理赛道**特别草台班子，基本上就是一个类似跑道的椭圆。

### 电路板

电路板需要统一在 嘉立创 为 大赛制定专门的比赛免费下单制作 ， 电路板上在正面覆铜层 放置队伍信息（同时包括学校名称、队伍名称、制作日期）， 在丝网层需要包括有制作文件链接二维码。

  - AURIX 系列 ： TC212、TC264、 TC297、 TC364、TC377、TC387（待补充）

  - Ai8051U，STC32G144K；（待补充）
  - STC32G12K128， STC8H8K64U， STC32F12K54；

3. NXP微控制器以及MicroPython开发板

  - 限定NXP公司微控制器，以及基于NXP控制器的MicroPython控制开发板。 
  - 主控芯片推荐使用 RT 系列单片机， 视觉模块使用 i.MXRT 系列和MCX Vision系列。

4. 龙芯中科开发板

  - 限定使用龙芯中科指定的微控制器对应的开发板。 第20届竞赛赠送的板卡依然可以使用。 开发工具除了使用 Linux系统之外， 还可以直接使用 C语言通过IDE下载工具完成软件的开发。

#### 双核支持
- TC264是双核处理器，项目通过`cpu0_main.c`和`cpu1_main.c`分别运行在两个CPU核心上
- `bsp_freertos_cpu0.c`负责CPU0上的FreeRTOS配置
- `zf_clock.c`也是只负责CPU0上的时钟，逐飞没做双核，我也没做 TODO

### 遇到的问题

- 原本打算用cpp写，发现他有cpp异常处理机制，很神秘啊，我把异常处理关了类内的函数也不能用。于是选择了纯c写。

- 很神秘啊，流缓冲区不能使用，我也不知道为啥。我最终选择了逐飞的fifo库，测了两次流缓冲区都不行，放弃了。

- 测试期间遇到了指针越界，内存不对齐去解指针，外设未初始化就调用，临界资源被同时访问。这些被卡死进错误的问题，大家多排查。

- 也遇到了舵机不能调整居中（认了），电机变化过快伞齿会卡住的问题（PID KP小一点 KD大一点 让他别那么大变化率）。

### 使用工程注意：

1. 工程导入到软件之后，每次增加文件都要选中工程然后点击refresh刷新一下之后再编译

2. 工程默认设置为关闭优化，可以自己右击工程选择properties->C/C++ Build->Setting，然后在右侧的窗口中找到C/C++ Compiler->Optimization->Optimization level处设置优化等级，一般默认新建立的工程都会默认开2级优化，因此大家也可以设置为2级优化

3. 对于TC系列默认是不支持中断嵌套的，希望支持中断嵌套需要在中断内使用 enableInterrupts(); 来开启中断嵌套

4. 所有的临界资源访问均没加锁，来提高访问速度。为了不加锁，牺牲了很多方便的操作。目前不加锁访问东西无问题。使用不当会导致临界资源被多个访问，会出问题
   - 串口3的发送功能，未保护，如果发送不当会卡死

### 任务/中断调度逻辑

#### FreeRTOS 任务（CPU0）

| 任务 | 优先级 | 周期 | 功能 |
|------|--------|------|------|
| `led_task` | 2 | 500ms | LED4 固定闪烁 |
| `key1_task` | 3 | 信号量触发 | 按键A→蜂鸣器100ms + LED1翻转 |
| `key2_task` | 3 | 信号量触发 | 按键B→蜂鸣器100ms + LED2翻转 |
| `key3_task` | 3 | 信号量触发 | 按键C→蜂鸣器100ms |
| `uart3_protocol_task` | 4 | 信号量触发 | 串口3协议解包→自动控制更新 |
| `test_task` | 5 | 100ms | 打印 R9DS 通道数据到串口0 用不到可注释|

#### 中断服务（CPU0）

| 中断 | 周期 | 功能 |
|------|------|------|
| `cc61_pit_ch1_isr` | **20ms** | 电机舵机控制中枢 — 根据 r9ds_ctrl 模式分发遥控/自动控制，运行电机PID |
| `cc61_pit_ch0_isr` | **1ms** | 按键扫描 + R9DS SBUS 帧解析（`device_r9ds_update`） |
| `uart0_rx_isr` | 事件触发 | 串口0（上位机通讯）接收中断 → FIFO |
| `uart1_rx_isr` | 事件触发 | 串口1（SBUS遥控器 100K/9E2）接收中断 → FIFO |
| `uart2_rx_isr` | 事件触发 | 串口2 接收中断 → FIFO |
| `uart3_rx_isr` | 事件触发 | 串口3 接收中断 → FIFO + 信号量给 `uart3_protocol_task` |

### R9DS 遥控器系统（SBUS 协议）

#### 硬件配置

| 参数 | 值 |
|------|-----|
| 接收机 | 乐迪AT9S R9DS |
| 协议 | Futaba SBUS |
| 串口 | UART1（`bsp_uart1`） |
| 波特率 | **100000** (100K) |
| 引脚 | TX=P15.0, RX=P15.1 |
| 帧格式 | 9-bit（8数据+1偶校验）、2停止位（`UART_CFG_9E2`） |
| 帧长度 | 25 字节 |
| 帧头/帧尾 | 0x0F ... 0x00 |
| 通道数 | 16 通道 × 11-bit |

#### 软件分层

| 层 | 文件 | 职责 |
|----|------|------|
| **BSP** | `bsp_uart.c/h` | UART1 硬件初始化（`uart_sbus_init`），中断接收 FIFO 缓冲 |
| **Device** | `device_r9ds.c/h` | SBUS 帧同步、16通道位提取、通道映射、在线检测（1s超时） |
| **Module** | `r9ds_ctrl.c/h` | 遥控器控制逻辑：模式判定、死区处理、速度/角度映射、S1/S2/S3 行为 |

#### 通道映射

| R9DS 成员 | SBUS 源 | 物理含义 | r9ds_ctrl 用途 |
|-----------|---------|----------|----------------|
| `rc.ch0` | SBUS_CH4 | 左摇杆水平 | 未使用 |
| `rc.ch1` | SBUS_CH3 | 左摇杆垂直 | **油门** — 前进/后退 |
| `rc.ch2` | SBUS_CH1 | 右摇杆水平 | **转向** — 左转/右转 |
| `rc.ch3` | SBUS_CH2 | 右摇杆垂直 | 未使用 |
| `rc.s1` | SBUS_CH10 | 2段开关 | **滑行(1) / 刹停(2)** |
| `rc.s2` | SBUS_CH9 | 2段开关 | **蜂鸣器关(1) / 开(2)** |
| `rc.s3` | SBUS_CH5 | 3段开关 | **手动(1) / 无控(2) / 自动(3)** |
| `rc.sw1~3` | SBUS_CH6~8 | VR旋钮 | 未使用 |

#### 控制参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 死区 | 980 ~ 1020 | 在此范围内视为中性"0" |
| CH1 油门范围 | 200 ~ 1800 | 映射到 ±150 cm/s（1000=停止） |
| CH2 转向范围 | 200 ~ 1800 | 映射到 ±35°（1000=直行） |
| 离线判定 | **1s 无有效帧** | 自动回退到 AUTO 模式 |

#### S1 / S2 / S3 开关行为

| 开关 | 值 | 行为 |
|------|-----|------|
| **S3** | 1 (上位) | **手动模式** — 遥控器接管，屏蔽自动控制 |
| | 2 (中位) | **无控制模式** — 电机立即刹停 + 舵机回中 |
| | 3 (下位) | **自动模式** — 串口协议接管，遥控器无效（默认） |
| **S1** | 1 | **滑行模式** — 油门回中时 PWM=0，电机自由减速 |
| | 2 | **刹停模式** — 油门回中时 PID 闭环制动到 0 |
| **S2** | 1 | 蜂鸣器关闭 |
| | 2 | 蜂鸣器持续鸣响 |

#### 在线检测机制

`device_r9ds_update()` 由 1ms ISR 驱动，每次调用递增 `offline_tick`。收到有效 SBUS 帧（帧头 0x0F + 帧尾 0x00 均匹配）时清零。

`device_r9ds_is_online()` 判定：`offline_tick < 1500`（即连续 1.5s 未收到有效帧=离线）。

离线时 `r9ds_ctrl_get_mode()` 强制返回 AUTO，确保遥控器断电后自动回退到安全状态。
